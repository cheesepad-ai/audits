#### üü† [H-1] Tax Conversion Routes to Arbitrary `USD1` Token (Liquidity/Price Control Risk)

**Description:**
Collected taxes are swapped from `BEPE` into `USD1` via a hardcoded route `[BEPE -> USD1]`, and the pair is created against `USD1` in the constructor. Using a non-bluechip token for tax conversion concentrates risk: if `USD1` is illiquid or controlled by insiders, conversions can occur at adverse or manipulated prices, effectively extracting more value than the nominal 2% tax.

```solidity
address public immutable USD1 = 0x8d0D000Ee44948FC98c9B98A4FA4921476f08B0d;

lpPair = IDexFactory(dexRouter.factory()).createPair(
    address(this),
    USD1
);

function swapTokensForUSD1(uint256 tokenAmt, address recipient) private {
    address[] memory path = new address[](2);
    path[0] = address(this);
    path[1] = USD1;

        tokenAmt,

        recipient,
        block.timestamp

}
```

**Impact:**
- If `USD1` liquidity is shallow or controlled, tax conversions can be executed at unfavorable rates, amplifying effective fees beyond 2%.
- Insiders can manipulate `USD1` pool pricing to capture value from conversions.
- Users bear hidden price impact through ongoing sell pressure into a potentially illiquid market.

**Location:**
